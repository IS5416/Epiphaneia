# 上线后 (Post-Launch)

## 目标

持续监控系统运行状态，收集用户反馈，驱动产品迭代优化。

## 输入

- 生产环境运行中的系统
- 用户反馈渠道

## 产出物

| 产出 | 文件名 | 说明 |
|------|--------|------|
| 监控仪表盘配置 | `monitoring/` | 关键指标看板（QPS、错误率、延迟、资源利用率） |
| 告警规则 | `alertRules.md` | 告警条件、通知方式、升级策略 |
| 用户反馈收集 | `userFeedback/` | 反馈汇总、分类、优先级 |
| 迭代规划 | `iterationPlan.md` | 下一版优先级排序的待办事项 |
| 故障复盘 | `postmortem/` | 事故时间线、根因分析、改进措施、时间节点 |
| 运营数据报告 | `opsReport.md` | 核心业务指标、用户增长、留存、转化 |

## 行为约束

1. **告警必响应**：P0/P1 告警必须 15 分钟内有人响应。
2. **无指责文化**：故障复盘关注流程改进而非追责。
3. **反馈闭环**：每条用户反馈必须在 7 个工作日内有回应（已修复 / 已排期 / 不采纳并说明原因）。
4. **数据驱动迭代**：next version 的功能优先级基于真实运营数据，非直觉。
5. **持续监控**：不因系统稳定而关闭监控，保持长期运行。
6. **安全维护**：关注依赖安全漏洞公告，定期升级补丁。

## 退出条件

本节点为持续运行态，无退出条件。当触发新一轮迭代时，回到 `ProductPlanning` 或 `IdeaValidation` 开始下一循环。
