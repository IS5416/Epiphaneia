# 架构设计 (Architecture Design)

## 目标

确定技术选型、系统架构、安全方案和部署架构，为开发提供清晰的技术蓝图。

## 输入

- `ProductPlanning/` 的产出物（PRD、领域模型、API设计、数据库设计）

## 产出物

| 产出 | 文件名 | 说明 |
|------|--------|------|
| 技术选型文档 | `techStack.md` | 框架/中间件/工具链选型及理由（必须基于 Java Spring 生态） |
| 系统架构图 | `systemArchitecture.md` | 模块划分、分层架构、服务拓扑、数据流 |
| 安全设计 | `securityDesign.md` | 认证授权方案、数据加密、OWASP Top 10 防护 |
| 部署架构 | `deploymentArchitecture.md` | 环境规划、容器化方案、CI/CD 流水线设计 |
| 项目骨架 | `projectScaffold.md` | 模块结构、包命名规范、基础配置项 |

## 行为约束

1. **技术栈硬约束**：基础框架必须是 Java Spring 生态（Spring Boot + Spring Cloud 或单体 Spring Boot）。
2. **选型需附理由**：每个技术选型必须说明为什么选它，以及考虑过的替代方案。
3. **安全优先**：安全设计必须覆盖认证、授权、输入校验、SQL注入防护、XSS防护、CSRF防护。
4. **可演进性**：架构设计需说明未来拆分为微服务的可能路径，但 MVP 阶段不做过度拆分。
5. **模块边界清晰**：明确各模块职责，禁止循环依赖。
6. **准备项目脚手架**：产出实际可构建的项目目录结构和 `pom.xml` / `build.gradle`。
7. **代码准入**：本阶段可以生成项目脚手架代码（空项目结构、基础配置），但不可写业务逻辑。

## 退出条件

- 全部产出物完成
- 技术选型得到用户确认
- 项目骨架可成功构建（`mvn compile` 或 `gradle build` 通过）
