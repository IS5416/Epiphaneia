# 想法验证 (Idea Validation)

## 目标

在写一行代码之前，验证产品想法的可行性和市场需求，确保值得投入开发资源。

## 输入

- 产品基础想法（由用户在对话中描述）

## 产出物

| 产出 | 文件名 | 说明 |
|------|--------|------|
| 市场调研报告 | `marketResearch.md` | 目标市场规模、增长趋势、市场痛点 |
| 竞品分析 | `competitorAnalysis.md` | 直接/间接竞品对比、优劣势、差异化机会 |
| 用户画像 | `userPersona.md` | 目标用户群体、行为特征、核心需求 |
| MVP范围定义 | `mvpScope.md` | 核心功能边界、不做的事项列表 |
| 验证结论 | `validationConclusion.md` | 最终判断：Go / No-Go / Pivot建议 |

## 行为约束

1. **禁止编码**：本阶段不产生任何代码。原型验证代码如需，单独标记并存放于 `prototype/` 子目录。
2. **数据驱动**：调研结论必须引用数据源，不可仅凭主观判断。
3. **诚实评估**：如果想法不可行（市场过小、竞品过强、需求伪命题），必须明确给出 No-Go 建议。
4. **MVP严格裁剪**：MVP范围只保留验证核心假设所需的最小功能集。
5. **等待用户确认**：产出物完成后，等待用户审查确认后再进入下一节点。

## 退出条件

- 全部产出物完成
- 用户明确确认进入 `ProductPlanning`
