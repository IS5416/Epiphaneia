## 阻断级问题 (BLOCKER — 必须修复)

### B-1: PromptTemplateManager 包路径与架构规范不匹配
**来源**：设计一致性审查
**文件**：`PromptTemplateManager.java` — 在 `internal/prompt/`，应为 `internal/llm/`
**设计依据**：systemArchitecture.md 明确 PromptTemplateManager 在 `internal.llm` 包
**修复**：移动到 `io.epiphaneia.agent.internal.llm`

### B-2: ActuatorProbeService 缺少 SSRF 防护
**来源**：设计一致性 + 架构安全审查
**文件**：`ActuatorProbeService.java:35`
**设计依据**：securityDesign.md §6.2 要求对 actuatorUrl 进行 SSRF 校验（禁止 file:// / localhost / 内网 IP）
**修复**：添加 URL 协议和 IP 范围校验

### B-3: analysis.txt 中 `{{question}}` 未包裹分隔符（Prompt Injection 风险）
**来源**：架构安全审查
**文件**：`prompts/analysis.txt:4`
**问题**：planning.txt 正确使用 `"""..."""` 包裹，analysis.txt 裸放用户输入
**修复**：用 `"""{{question}}"""` 包裹

### B-4: PrometheusQueryBuilder 参数 step/start/end 未使用（死参数 + 查询错误）
**来源**：测试质量审查
**文件**：`PrometheusQueryBuilder.java:22`
**问题**：`buildRangeQuery` 接受 step/start/end 但完全不使用，生成的 PromQL 缺少 `[range]`
**修复**：移除死参数，或实现 range 输出；重命名方法为 `buildAggregatedQuery`

### B-5: EsQueryBuilder.escape() 无 null 检查，buildErrorLogQuery 中 NPE
**来源**：测试质量审查
**文件**：`EsQueryBuilder.java:77`
**问题**：`escape(null)` → NPE。`buildErrorLogQuery` 传递 service 不检查 null
**修复**：`if (s == null) return "";`

### B-6: PromptTemplateManager 重叠占位符 bug
**来源**：测试质量审查
**文件**：`PromptTemplateManager.java:28-35`
**问题**：`String.replace()` 顺序依赖，`{{name}}` 和 `{{nameExtra}}` 同时存在时后者被部分替换
**修复**：按 key 长度降序排序后替换，或用正则 word-boundary 匹配

### B-7: MetricsQueryServiceImpl + LogQueryServiceImpl 零测试
**来源**：测试质量审查
**文件**：两个 ServiceImpl
**修复**：添加基础单元测试验证 DI 和 query() 返回非 null
