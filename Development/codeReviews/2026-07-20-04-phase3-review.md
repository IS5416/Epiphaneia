审查范围：commits 8087d20, ecac0a7, 3f0deaa (branch: feat/phase3-server)
审查维度：设计一致性、架构安全、测试质量

## 阻断级问题 (BLOCKER — 必须修复)

### B-1: POST /auth/tokens 使用 @RequestParam 而非 @RequestBody
**来源**：设计一致性审查
**文件**：`AuthController.java:84`
**问题**：apiDesign 规定请求体为 JSON `{"name":"CI pipeline"}`，但代码用 `@RequestParam String name` 从查询参数读取。
**修复**：改为 `@Valid @RequestBody CreateTokenRequest` DTO。

### B-2: ApiTokenResponse 字段命名 prefix→应为 token
**来源**：设计一致性审查
**文件**：`AuthController.java:99`
**问题**：创建 Token 返回 `ApiTokenResponse(..., rawToken, ...)` 但 DTO 字段名为 prefix，apiDesign 要求 `"token":"epi_..."`。
**修复**：ApiTokenResponse 添加 token 字段，或使用独立 CreateApiTokenResponse。

### B-3: POST /auth/login 返回 400 而非 401
**来源**：设计一致性审查
**文件**：`AuthController.java:44-47`
**问题**：登录失败抛 IllegalArgumentException→GlobalExceptionHandler 映射为 400。apiDesign 要求 401 + INVALID_CREDENTIALS。
**修复**：使用 @ResponseStatus(HttpStatus.UNAUTHORIZED) 自定义异常。

### B-4: 查询参数 applicationId → 应为 appId
**来源**：设计一致性审查
**文件**：`ConversationController.java:45`
**问题**：apiDesign 规定 `?appId={id}`，代码使用 `applicationId`。
**修复**：`@RequestParam(name = "appId", required = false)`。

### B-5: LlmController 导入 agent-core/internal 包
**来源**：架构安全审查
**文件**：`LlmController.java:5`
**问题**：导入 `io.epiphaneia.agent.internal.llm.ModelRouter` 违反包边界规则（仅 DiagnosisSkill 可跨边界）。
**修复**：ModelRouter.validateProvider() 移到公共 API 接口或通过 Skill 层调用。

### B-6: SSE 流式输出损坏——同步诊断阻塞发射器返回
**来源**：架构安全审查
**文件**：`ConversationController.java:92-115`
**问题**：`diagnosisSkill.diagnose()` 同步阻塞，所有 SSE 事件在方法返回前被缓冲。客户端收不到实时事件。
**修复**：使用 @Async 或显式异步执行诊断。

### B-7: ApiResponse.error() 构造函数参数不匹配
**来源**：架构安全审查
**文件**：`ApiResponse.java:13` + `ErrorDetail.java`
**问题**：ErrorDetail 有 3 参数 (code, message, details)，但 ApiResponse.error() 只传 2 个。clean compile 会失败。
**修复**：添加 ErrorDetail(code, message) 辅助构造函数（details 可为 null）。

### B-8: Conversation 端点缺少授权检查
**来源**：架构安全审查
**文件**：`ConversationController.java`
**问题**：所有 CRUD 操作仅验证认证状态，未验证所有权。单管理员场景缓解，多用户时成为漏洞。
**修复**：ponytail 标记 `// ponytail: single-admin — ownership check deferred`。

### B-9: Phase 3 零测试（47 文件无新增测试）
**来源**：测试质量审查
**修复**：新增 7 个关键测试文件：AuthControllerTest、SecurityConfigTest、GlobalExceptionHandlerTest、SessionAuthFilterTest。

## 高危问题 (HIGH)

- H-1: GlobalExceptionHandler 将 404 映射为 400（IllegalArgumentException 统一处理）
- H-2: GlobalExceptionHandler 缺 401/403/409 专用处理器
- H-3: CSRF 完全禁用（应为 Cookie 启用 + Bearer 豁免）
- H-4: CORS 未配置（Web UI 跨域请求被浏览器阻止）
- H-5: CRUD 控制器缺授权检查
- H-6: BearerToken 主体为字符串 "admin" 而非 Admin 对象（与 SessionAuthFilter 不一致）
- H-7: SSE 事件负载与 apiDesign §9 不一致（缺 timestamp、tokenCount、字段命名）
- H-8: AuthController Token 端点缺管理员上下文过滤

## 中优先级 (MEDIUM)

- M-1: Session Cookie 缺 Secure 标记
- M-2: RateLimitFilter 信任 X-Forwarded-For 无校验
- M-3: 密码复杂度未强制（≥8 位 + 大小写+数字）
- M-4: SSE error 事件泄露异常消息
- M-5: DELETE 端点返回 204 空体 vs apiDesign 期望 200 + message
- M-6: DataSource POST 缺重复 type 检查（409）
- M-7: Application probe 返回 200 → 应为 202
- M-8: SystemStatus 格式与 apiDesign 不匹配（缺 checks 对象）
- M-9: ChangePasswordRequest 缺密码规则校验
- M-10: RateLimitFilter ConcurrentHashMap 无界增长（内存泄露）
- M-11: DiagnosisSkill.hasActiveDiagnosis 全量加载消息

## 低优先级 (LOW)

- L-1: 初始密码持久化至日志归档（运维操作文档需覆盖）
- L-2: 缺安全头（X-Content-Type-Options、X-Frame-Options、HSTS）
- L-3: ConversationResponse 缺 messageCount/lastState
- L-4: 无分页支持
- L-5: tags String vs JSON 数组不一致
